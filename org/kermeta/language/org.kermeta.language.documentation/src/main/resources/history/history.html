Kermeta 2.1.0 (17/07/2013)

New features

  • Creation of a new project kermeta from an ecore file with options for the generation of the new project : Aspect programming, Visitor (Object view), user defined
  • A new kermeta Sample is availabe : "FSM". It is projects which allow to build a FMS (Finite State Machine) DSML with Kermeta 2
  • A new kermeta Sample is availabe : "Logo". It is projects which allow to build a Logo DSML with Kermeta 2

Improved features

  • Scala compilation error messages go to the default console instead of the build console (feature request #15754).
  • Kp syntax has been changed for support of the notion of modeltype. Modeltypes can be imported or extended. It is now clearer about the notion of resource and import.

Bug fixed

  • #15753; Navigating the outline sometimes raises a null argument error
  • #15955; StdIO.writeln now use the messagingsystem but display useless additonal info
  • #15960; uri of kind "platform:/resource/" doesn't correctly work for looking for bytecode in eclipse project Work on OS Windows and Linux
